1. O que é o Cur10usX

O Cur10usX é uma plataforma de gestão escolar moderna, pensada para o contexto angolano. Centraliza a gestão de alunos, professores, notas, assiduidade e comunicação numa única plataforma acessível de qualquer dispositivo.

Problema

Muitas escolas em Angola ainda gerem matrículas, notas e assiduidade em papel ou em ficheiros Excel dispersos. A comunicação entre escola, professores e encarregados é fragmentada e lenta.

Solução

O Cur10usX oferece uma plataforma unificada onde cada interveniente da comunidade escolar — gestor, professor, aluno e encarregado — tem acesso à informação relevante para o seu papel, em tempo real.

Características Principais

Multi-Escola

Cada escola é um tenant isolado com dados, utilizadores e configurações independentes.

Modular

17 módulos activáveis por escola. Cada instituição liga apenas o que precisa.

Responsivo

Funciona em computador, tablet e telemóvel. Dark mode incluído.

Seguro

Autenticação robusta, permissões granulares, dados encriptados e isolados por escola.

3. Funcionalidades por Perfil

Cada perfil tem acesso a funcionalidades específicas para o seu papel.

Gestor Escolar

  • Dashboard — Stat cards personalizáveis (alunos, professores, turmas, média geral, etc.), gráficos de distribuição e assiduidade
  • Gestão completa — CRUD de alunos, professores, encarregados, turmas, disciplinas, cursos
  • Solicitações — Aprovar/rejeitar candidaturas de matrícula online
  • Importação em massa — Upload de Excel/CSV para criar alunos, professores ou encarregados em lote (até 500 por ficheiro)

  • Agenda — Calendário semanal com horário de aulas
  • Notas — Lançar notas por trimestre (provas, tarefas, trabalhos, participação)
  • Assiduidade — Registar presenças, ausências e atrasos por aula
  • Tarefas — Criar tarefas com prazo, avaliar submissões

Aluno

  • Notas — Consultar todas as notas por disciplina e trimestre
  • Assiduidade — Ver presenças, faltas e percentagem
  • Tarefas — Ver tarefas pendentes, submeter trabalhos

Encarregado de Educação

  • Dashboard do educando — Mesmo dashboard de progresso do aluno, com selector de filho(a)
  • Notas e assiduidade — Acompanhamento em tempo real
  • Comunicação — Mensagens internas com a escola

4. Fluxo de Onboarding

Do registo da escola à utilização diária — 4 passos simples.

1

Registo da Escola

O responsável preenche o formulário com dados da escola (nome, NIF, email, telefone, cidade, província) e cria a conta de administrador.

2

Aprovação

O super admin da plataforma analisa e aprova o registo. A escola passa ao estado "ativa" e o admin recebe notificação.

3

Configuração

O gestor escolar configura: cria turmas, disciplinas e cursos; adiciona professores e alunos (individualmente ou via importação em massa).

4

Utilização

Professores lançam notas e assiduidade. Alunos e encarregados acompanham o progresso nos seus dashboards.

Candidaturas Online

Candidatos externos podem solicitar matrícula sem necessitar de conta:

  1. Aceder a /aplicacao e seleccionar a escola
  2. Preencher formulário com dados pessoais e académicos
  3. Receber código de acompanhamento
  4. Verificar estado em /aplicacao/status
  5. Se aprovada, a escola procede à matrícula directamente na plataforma

10. Autenticação e Permissões

NextAuth v5 com JWT strategy, múltiplos providers e sistema de permissões granulares.

Providers

  • Credentials — Email + password (bcryptjs hash, 10 rounds)
  • Google — OAuth 2.0 (opcional, controlado por GOOGLE_AUTH_ENABLED)

JWT Token

O token JWT contém toda a informação necessária para autorização, evitando queries adicionais:

CampoDescrição
idUser ID
roleRole do utilizador (super_admin, school_admin, etc.)
schoolId, schoolSlugEscola associada
adminLevelprimary ou secondary (para school_admin)
permissionsArray de permissões granulares
schoolFeaturesFeatures activas da escola (para UI)
mustChangePasswordFlag para forçar alteração de password

Funções de Autorização

FunçãoUso
requireRole(roles[])Verifica se o user tem uma das roles permitidas
requirePermission(roles[], permission?)Verifica role + permissão granular para admins secundários
getSchoolId(session)Extrai schoolId da sessão para filtragem multi-tenant

Fluxo de Password

  • Importação em massa gera passwords temporárias (6 bytes random, base64url)
  • Flag mustChangePassword activa — gate component bloqueia acesso até alteração
  • Forgot password envia token por email (via Resend), válido por 1 hora

11. Sistema de Feature Flags

Cada escola tem um conjunto de features activáveis, armazenado como JSON no modelo School.

Como Funciona

  1. O campo School.features armazena um objecto JSON: { "students": true, "finances": false, ... }
  2. Na criação da escola, as features padrão são activadas automaticamente
  3. O gestor escolar pode activar/desactivar módulos nas configurações
  4. As features são propagadas para o JWT token na sessão (schoolFeatures)
  5. O menu e componentes verificam isFeatureEnabled() antes de renderizar

Integração com Menu

O ficheiro src/lib/features.ts define o mapeamento entre features e items de menu. Quando uma feature é desactivada, os items de menu correspondentes ficam invisíveis para todos os utilizadores dessa escola.
